difyAgentRunner模块解析

Dify 后端 Agent 模块详细分析

1. 模块架构与功能概述

Agent 模块是 Dify 后端的核心功能之一，负责实现智能代理的核心逻辑，支持多种推理策略和工具集成。该模块采用了清晰的分层设计，包括实体定义、运行器实现、输出解析和策略管理等组件。

核心功能

  • 多种推理策略：支持思维链（Chain of Thought）和函数调用（Function Calling）两种核心策略
  • 工具集成：无缝集成外部工具和数据集检索功能
  • 迭代执行：通过多轮交互逐步解决复杂问题
  • 流式输出：实时返回模型生成的内容，提升用户体验
  • 历史管理：维护对话历史，提供上下文理解能力

2. 核心实体定义

AgentEntity

解析：

  • Strategy：定义了两种核心策略
    • CHAIN_OF_THOUGHT：思维链策略，通过多步推理解决问题
    • FUNCTION_CALLING：函数调用策略，直接调用工具获取信息
  • provider：AI 服务提供商（如 OpenAI、Anthropic）
  • model：模型名称（如 gpt-4、claude-3）
  • prompt：提示模板配置，用于引导模型生成特定格式的响应
  • tools：工具列表，定义了 Agent 可以使用的外部工具
  • max_iteration：最大迭代次数，防止无限循环

3. 核心运行器实现

3.1 BaseAgentRunner

BaseAgentRunner 是所有 Agent 运行器的基类，提供了通用的功能和方法。

核心功能:

  • 初始化 Agent 运行环境和工具
  • 管理 Agent 思考历史
  • 处理工具调用和结果
  • 组织提示消息

这个方法可以用来初始化agent,同时他会调用自己的_convert_tool_to_prompt_message_tool方法去转化格式获取实例，让llm能够理解，然后我们把这个实例和propmt存起来，初始化就完成了，同时单独处理数据集工具，做到工具的分离

create_agent_thought() 这个函数会记录当前这次对话的思考全流程，然后返回记录到数据库里后返回的id，save_agent_thought() 负责更新这个记录。

organize_agent_history()从数据库提取历史对话记录，并将其重组为 LLM 可理解的 PromptMessage 列表

总结

BaseAgentRunner 通过”初始化 → 调用 → 存储 → 记忆”的闭环，构建了 Agent 的最小可运行单元

3.2 CotAgentRunner

CotAgentRunner 实现了思维链（Chain of Thought）策略，是 Agent 模块的核心运行器之一。

核心功能：

  • 实现思维链推理过程
  • 处理模型输出的流式解析
  • 管理工具调用和结果处理
  • 控制迭代执行流程

核心概念： ReAct模式：

ReAct = Reasoning（推理）+ Acting（行动），我们重复Thought（思考）→ Action（行动）→ Observation（观察）这个循环，直到得到最终答案

AgentScratchpadUnit

AgentScratchpadUnit = ReAct 单轮迭代的数据容器，保存 LLM 的原始响应、思考内容、行动指令、工具返回结果

流程逻辑

这个是最主要的方法，我们通过run方法里面的while循环来实现ReAct模式

首先，我们要初始化run方法，我们先创建一个实体，然后初始化他的配置，给他设定循环上限，最后给他定义函数，这样，他才能执行后面的循环。

进入循环，我们先初始化本轮配置，默认本轮结束了，如果是最后一轮，清空工具配置。然后我们重新创建本次对话的记录，如果不是首轮，就通知前端，然后我们剩余可用 token，防止超长。

然后我们调用大模型，对答案进行流失化处理 CotAgentOutputParser.handle_react_stream_output，随后，我们创建这一次的记忆单元AgentScratchpadUnit，用来记录完整的ReAct流程，然后我们通知首轮前端

流式处理输出，然后把本轮ReAct结果保存

进行检查，然后计算token消耗，然后保存本次对话记录

最后是判断循环的条件，处理可能进行的动态调用，以及更新工具参数和步数

完整流程图

这个 mermaid 流程图描述了一个用户与 ReAct 模型进行对话的过程。用户向 ReAct 模型提问，ReAct 模型会生成一个 Prompt，然后开始循环，直到生成 Final Answer 或者工具调用。在每次循环中，ReAct 模型会调用 LLM，并获取 Thought 和 Action。如果 Action 是 Final Answer，则结束循环；否则，执行工具调用，并保存结果，然后继续下一轮。最后，ReAct 模型保存最终结果，并返回答案给用户。
